Pulseboard alerts monitor activation drop-offs, engagement decline, and sudden workspace inactivity. Each alert can post into Slack, email, or an internal webhook receiver.

This guide is intentionally operational. Real buyers want the docs to work for non-technical team members too, because support, product, and growth all read the same help center.

A useful documentation service does not overexplain. It gives the user just enough context to make the setting correctly on the first try.

## When to use alerts

Alerts are best for:

- **Activation drop** - Notify when new user activation falls below threshold
- **Engagement decline** - Alert when daily active users drop significantly
- **Workspace inactivity** - Flag accounts with no activity in 30+ days
- **Trial conversion** - Notify when trial users are about to expire

## Set up an alert

<Steps>

1. Choose the metric you want to monitor
2. Define the threshold window (daily, weekly, or custom)
3. Pick a destination: Slack, email, or webhook
4. Set a cooldown period to prevent alert fatigue
5. Link to a saved segment or dashboard for follow-up

</Steps>

## Destination options

### Slack

Send alerts to a Slack channel or direct message:

```json
POST /v1/alerts/test
{
  "workspace_id": "ws_2048",
  "destination": "slack",
  "threshold_type": "weekly_retention_drop",
  "threshold_percent": 12
}
```

### Email

Route alerts to team inboxes with customizable templates.

### Webhook

Send alerts to your own systems for custom processing:

```json
{
  "workspace_id": "ws_2048",
  "risk_score": 78,
  "changed_from": 54
}
```

<Callout type="warning" title="Cooldown settings">
  Set a cooldown so one incident does not flood your team. We recommend at least 1 hour between repeated alerts.
</Callout>
