Pulseboard supports two installation methods: a no-build browser snippet for simple products and a React wrapper for teams that want typed event helpers and route-aware tracking.

The docs service angle matters here because technical documentation has to be executable. Good docs reduce ambiguity in the first five minutes of product setup.

In a real client project, this page would be matched to the product stack and include copy-paste implementation blocks for the exact framework in use.

## Choose the install path

Select the method that best fits your tech stack:

- **Browser snippet** - Pure JavaScript, works with any framework, no build step required
- **React SDK** - TypeScript support, route-aware tracking, React-specific utilities

## Browser snippet

Add the following script to your HTML `<head>` or root layout:

```html
<script>
  window.pulseboard = window.pulseboard || [];
  window.pulseboard.push(["init", { workspaceKey: "pk_live_demo123" }]);
  window.pulseboard.push(["track", "user_signed_up", { plan: "starter" }]);
</script>
```

<Callout type="warning" title="Security note">
  Always use your public workspace key (starts with `pk_`) for browser-side tracking. Never expose private keys in client-side code.
</Callout>

## React wrapper

For React applications, install the SDK:

```bash
npm install @pulseboard/react
```

Then initialize in your app:

```tsx
import { PulseboardProvider, useTrack } from '@pulseboard/react';

function App() {
  return (
    <PulseboardProvider workspaceKey="pk_live_demo123">
      <YourApp />
    </PulseboardProvider>
  );
}

function SignupForm() {
  const track = useTrack();
  
  const handleSubmit = () => {
    track("user_signed_up", { plan: "starter" });
  };
  
  return <button onClick={handleSubmit}>Sign up</button>;
}
```

## Verify events

After installation, verify events are flowing:

1. Open your application in development mode
2. Trigger the events you want to track
3. Go to **Live Stream** in Pulseboard dashboard
4. Confirm events appear within a few seconds

<Steps>

1. Copy your workspace public key from Settings → API Keys
2. Paste the snippet into the root layout or document head
3. Send one test event from staging and confirm it appears in Live Stream
4. Promote the same setup to production after verification

</Steps>

## Troubleshooting

If events don't appear:

- Check browser console for errors
- Verify the workspace key is correct
- Ensure the script loads before your tracking calls
- Check that your domain is added to allowed origins in settings
