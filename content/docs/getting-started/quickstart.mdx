Pulseboard is a fictional product analytics SaaS for product-led teams. It tracks activation, weekly retention, churn signals, and workspace-level product health across web apps and internal tools.

This demo page is written the way a real launch-stage help center should be written: short introduction, practical setup sequence, concrete outcomes, and enough product context that a user can act without asking support for basic clarification.

A founder buying docs-as-a-service usually wants exactly this kind of page first. It lowers support volume, gives new users confidence, and makes the product feel complete.

## What Pulseboard does

Pulseboard helps product teams understand user behavior through event tracking and visualization. It provides:

- **Activation tracking** - Measure when users reach their first "aha" moment
- **Retention analysis** - Understand weekly and monthly retention patterns  
- **Churn detection** - Identify at-risk accounts before they leave
- **Workspace health** - Monitor overall product engagement

## Before you begin

Before starting the setup, ensure you have:

- A Pulseboard account (sign up at pulseboard.example.com)
- Access to your web application codebase
- Permission to add tracking code to your application

## Setup checklist

<Steps>

1. **Create a new workspace** - Sign in to Pulseboard and create a new workspace. Add your production URL in the workspace settings.

2. **Install the SDK** - Add the browser snippet or React SDK to your main app shell. See [Install the tracking snippet](/demo/pulseboard/installing) for detailed instructions.

3. **Track baseline events** - Send three core events:
   - `user_signed_up` - When a user completes registration
   - `workspace_created` - When a new workspace is provisioned
   - `feature_activated` - When a user first uses a key feature

4. **Invite your team** - Add at least one teammate and pin the default Activation dashboard to get started quickly.

</Steps>

## Expected result

After completing these steps, you'll have:

- A functioning Pulseboard workspace with your app connected
- Three core events flowing into the dashboard
- An Activation dashboard showing user onboarding progress
- A shared view your entire team can access

<Callout type="info" title="Pro tip">
  You do not need a full data warehouse to get value from the product. One clean event taxonomy is better than 50 noisy events.
</Callout>

## Key points

- **Start simple** - Most teams get a usable dashboard from three to five events on day one
- **Focus on activation** - The first question to answer is: "When do users reach first value?"
- **Iterate over time** - Add more events as your product evolves and your questions become more specific
